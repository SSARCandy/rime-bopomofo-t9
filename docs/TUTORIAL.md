# 倉輸入法 T9 注音鍵盤：從零開始的完整教學

這份教學是為「完全沒有程式背景」的新手準備的。讀完這篇，你就會知道這個九宮格（T9）注音鍵盤是怎麼運作的，以及如果想改成自己喜歡的樣子，該去改哪些檔案。

## 1. 這是什麼？運作原理是什麼？

要了解這個鍵盤怎麼運作，你可以把它想像成分工合作的三個部門：

1. **外觀部（Hamster 倉輸入法 App）**：負責顯示鍵盤在你的手機螢幕上。當你按下滑動、長按某個按鍵時，它負責告訴大腦「主人按了什麼」。
2. **大腦部（RIME 引擎規則）**：負責解讀外觀部傳來的訊號。例如，你按了數字 `1`，大腦知道 `1` 代表可能是 `ㄅ`、`ㄉ` 或 `ㄚ`。
3. **記憶部（詞庫字典）**：大腦根據推測出的注音，去翻找記憶裡的字典，找出對應的中文字，然後顯示在螢幕上讓你選。

---

## 2. 三個核心檔案介紹

在你的資料夾裡，有三個最重要的檔案，分別對應上面的三個部門：

1. **`t9bopomo.yaml`**（外觀部）：控制鍵盤長什麼樣子、按鈕在哪裡、長按會出現什麼、滑動會輸入什麼。
2. **`bopomofo_t9.schema.yaml`**（大腦部）：定義了「按鍵對應規則」（例如按 1 代表什麼注音），以及聲調的設定。
3. **`terra_pinyin.dict.yaml`**（記憶部）：這是一本超大的字典，裡面記錄了每個中文字的拼音/注音。

## 3. 如果我想改「鍵盤畫面與按鍵功能」，怎麼改？

**你要改的檔案：`t9bopomo.yaml`**

這個檔案是用一種叫做 YAML 的格式寫的。不要怕，它其實就是條列式的設定。
在這個檔案裡，鍵盤是「一排一排（rows）」設定的，然後每一排裡面有「按鍵（keys）」。

### 常見修改 1：更改滑動（Swipe）輸入的符號
打開檔案，你會看到類似這樣的區塊：
```yaml
          - action: { character: { char: "1" } }
            label: "1\nㄅㄉㄚ"
            swipe:
              - direction: up
                action: { symbol: { char: "！" } }
```
- `label: "1\nㄅㄉㄚ"`：這是印在按鈕上的字。`\n` 是換行的意思。
- `direction: up`：意思是「向上滑動」。
- `action: { symbol: { char: "！" } }`：意思是「直接在螢幕上打出驚嘆號（不過 RIME，直接打字）」。

**👉 怎麼改？**
如果你想把向上滑動改成問號，只要把 `"！"` 改成 `"？"` 存檔就好。

### 常見修改 2：更改長按彈出的選單（Callout）
長按按鍵會彈出一個小氣泡，這叫 `callout`。
```yaml
            callout:
              - label: "ㄚ"
                action: { character: { char: "1" } }
              - label: "ㄉ"
                action: { character: { char: "d" } }
              - label: "ㄅ"
                action: { character: { char: "b" } }
```
- `label` 是氣泡上顯示的字。
- `action` 裡面的 `char` 是「傳送給大腦的秘密代碼」。
為什麼 `ㄉ` 是傳 `d`？因為這能告訴大腦「我很確定我要輸入『ㄉ』，不是模糊的數字 1」。如果你只傳 `1`（像 `ㄚ` 那樣），大腦就要自己猜這到底是什麼。

**注意：** iOS 鍵盤右半邊的按鍵，長按氣泡會往左長，所以為了讓視覺順序是由左到右，代碼裡的排列順序必須是**反過來的**。

## 4. 如果我想改「按鍵對應的注音」，怎麼改？

**你要改的檔案：`bopomofo_t9.schema.yaml`**

有時候你可能覺得「我不想把 ㄅㄉㄚ 放在按鍵 1，我想把 ㄅㄆㄇㄈ 放在一起」。這時候就需要改大腦的規則。

在這個檔案裡，找到 `speller:` 下面的 `algebra:` 區塊。這裡充滿了看起來像亂碼的替換規則：
```yaml
    # 這是把明確的代碼（如 b 代表 ㄅ）換成數字 1，這樣你按 1 引擎就知道可能包含 b
    - derive/b/1/
    - derive/d/1/
    - derive/a/1/
```
這裡的邏輯是：
1. 字典檔本來是用類似英文的代碼記錄字的讀音。我們用特定的英文字母來代表注音符號（例如 `b` 就是 `ㄅ`）。
2. 然後我們告訴大腦（RIME）：「只要字典裡的發音有 `b`、`d` 或 `a`，通通把它當作你按了鍵盤的 `1`。」

**👉 怎麼改？**
假設你想把 `ㄆ` (代表字母是 `p`) 從按鍵 `4` 移到按鍵 `1`，你需要做兩件事：
1. 在這個檔案的 `algebra:` 裡找到 `- derive/p/4/`，把它改成 `- derive/p/1/`。
2. 回到外觀檔 `t9bopomo.yaml`，把按鍵 1 跟按鍵 4 上的字（label）改對應，以免畫面上寫 ㄅㄉㄚ 但其實打出來有 ㄆ。

## 5. 如果我想改「聲調」，怎麼改？

聲調的設定同樣在 **`bopomofo_t9.schema.yaml`**。
為了讓系統能精準識別，我們借用了四個不會跟注音混淆的英文字母來當作聲調鍵：
- 一聲：`q`
- 二聲：`w`
- 三聲：`x`
- 四聲：`y`

在 `t9bopomo.yaml` 中，你可以看到一個專門打聲調的按鍵，裡面定義了長按呼出：
```yaml
            callout:
              - label: "ˋ"
                action: { character: { char: "y" } }
              - label: "ˇ"
                action: { character: { char: "x" } }
...
```
如果你想把長按的聲調符號換位置，或換成別的觸發字母，就必須同時修改這兩個檔案。不過一般來說，這部分的設定已經是配合引擎的最佳化了，不建議隨便更動。

---

## 6. 如果我想加自己的詞庫，怎麼加？

**你要改的檔案：`terra_pinyin.dict.yaml`（或者是新建一個自訂詞庫檔）**

打開字典檔，拉到最下面，你會看到類似這樣的格式：
```text
(中文字)   (發音代碼)   (權重/優先順序)
你好      n i h K    100
```
*(注意：中間一定要用鍵盤上的 `Tab` 鍵隔開，不能用空白鍵！)*

發音代碼對應表可以參考專案裡的 `BOPOMOFO_MAPPING.md` 說明。你可以自己加入常用的詞彙，並給予較高的權重數字，這樣打字時它就會優先出現。

---

## 7. 我改完了，要怎麼放進手機裡使用？

1. **上傳檔案**：
   - 如果你有 iCloud：把改好的這三個檔案丟進 iCloud 雲碟裡的 `Hamster/RIME/Rime` 資料夾。
   - 如果用 Wi-Fi：打開手機倉輸入法 App，進入「Wi-Fi 傳檔」，用電腦瀏覽器連上顯示的網址，把檔案拖進去。
2. **重新部署（最重要的一步！）**：
   檔案放進去後，**絕對不會立刻生效**。
   你必須打開倉輸入法 App，進入 **RIME** 選單，點擊 **「重新部署」**。
   這就像是叫大腦「重新讀取所有規則跟設定檔」。看到「部署成功」的提示後，你的新設定就會生效了！

---

## 總結流程

1. 想改畫面？ → 去改 `t9bopomo.yaml`
2. 想改注音分佈、聲調邏輯？ → 去改 `bopomofo_t9.schema.yaml`
3. 想加新詞？ → 去改 `terra_pinyin.dict.yaml`
4. 改完記得放進手機並**重新部署**！

慢慢摸索，修改前記得先把原本的檔案備份（複製一份備用，或是壓縮起來），就算改壞了也能蓋回來。祝你玩得開心！
